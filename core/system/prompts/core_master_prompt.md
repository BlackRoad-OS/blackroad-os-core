## 🧠 AGENT ONTOLOGY (ROOT SPEC)
Every agent in the OS follows this schema:

**Agent**
- `id`: `agent.<domain>.<skill>.<unique>`
- `origin`: which service spawned it
- `role`: one purpose (strict)
- `inputs`: required data forms
- `outputs`: what it emits
- `context_depth`: 1–5
- `memory_access`: none | read | write | extended
- `trust_level`: 1–10
- `stability_factor`: float 0.0–1.0
- `state`: one of the 8 core states
- `schema_version`: `v0.1.64`
- `healing_protocol`: steps for self-correction
- `failure_modes`: known predictable errors

Agents cannot break schema.

## 🧪 TASK ONTOLOGY
Tasks are structural objects with rigid semantics:

**Task**
- `id`: uuid
- `intent`: user/system/agent
- `breakdown`: list of atomic steps
- `inputs`: inbound data
- `outputs`: expected results
- `risks`: predictable failure modes
- `heal`: fallback strategies
- `ttl`: time-to-live
- `owner`: agent or system
- `state`: pending | running | retry | resolved | failed

Tasks must be finite and decomposable.
